# Маяк UI kit & styleguide

Финальная страница «Маяк» становится визуальным эталоном всего приложения. Все этапы онбординга должны выглядеть как часть одного спокойного личного пространства, а не как разные продукты.

## Принципы

1. **Viewport-first.** Каждый этап должен помещаться в один экран без page-scroll на desktop. Если данных много, скроллится конкретная карточка, а не вся страница.
2. **Спокойный проводник.** Маяк помогает новичку пройти путь без давления. Тексты объясняют, зачем нужен шаг, и снимают тревожность.
3. **Один понятный шаг.** На каждом экране есть один главный CTA и короткое пояснение, что будет дальше.
4. **Мягкая технологичность.** Aurora-фон, blur, rounded-3xl, пульсирующий активный шаг и карты знаний создают ощущение AI-помощника, но не перегружают интерфейс.
5. **Единые состояния.** `primary` — активное/готовое, `secondary` — спокойная подложка, `accent` — тема в работе, `muted` — впереди/нейтрально, `deep` — ключевой summary/hero.

## Токены

Токены живут в `app/globals.css` и используются через Tailwind v4 utilities:

- `background`, `foreground`, `card`, `muted`, `border`, `ring` — базовая система;
- `primary`, `secondary`, `accent` — действия, статусы и мягкие выделения;
- `aurora-1`, `aurora-2`, `aurora-3` — фоновые световые пятна;
- `deep`, `deep-foreground`, `deep-muted`, `deep-border` — иммерсивные summary-панели;
- `--shadow-card`, `--shadow-soft` — мягкие тени без тяжёлого enterprise-вида.

## Компоненты

UI kit находится в `src/shared/ui/mayak`.

- `MayakShell` — единая оболочка страницы с aurora-фоном, `h-screen`, фиксированной рабочей областью и контентной сеткой `max-w-6xl`.
- `MayakTopBar` — бренд «Маяк», контекст этапа, пользователь.
- `MayakPanel` — основная карточка/секция; вариант `deep` использовать для hero и summary.
- `MayakSectionHeader` — kicker, H1 и короткое объяснение.
- `MayakButton`, `MayakLinkButton`, `MayakActionBar` — действия.
- `MayakField`, `MayakInput`, `MayakOptionCard` — формы и выбор.
- `MayakBadge`, `MayakIconBadge`, `MayakStatCard`, `MayakProgressBar`, `MayakProgressRing` — статусы, метрики, прогресс.

## Правила применения

- Каждый новый экран начинается с `MayakShell` и `MayakTopBar` либо находится внутри уже существующего shell.
- Экранный контейнер строится как `h-full min-h-0`, а рабочая зона — как `flex-1` или CSS grid с `min-h-0`.
- Основной контент — только через `MayakPanel`/`MayakOptionCard`, без локальных кастомных поверхностей.
- Длинные списки, таблицы и чаты должны иметь внутренний `overflow-y-auto`; внешний page-scroll запрещён для основного desktop flow.
- Главная кнопка на экране — `variant="primary"`, вторичные — `secondary` или `ghost`.
- Карточки этапов и ответы диагностики используют одинаковое selected-состояние: `border-primary/60 bg-primary/5`.
- Служебные подписи оформляются как `font-mono uppercase tracking-wider`.
- Новые цвета, тени и радиусы добавляются сначала в styleguide, потом в код.

## Не делать

- Не возвращаться к старому CSS-module виду для отдельных шагов.
- Не делать плотные таблицы без карточной адаптации и внутреннего scroll-контейнера.
- Не использовать агрессивный красный для обычных пробелов знаний: это «зона поддержки», а не ошибка.
- Не писать тексты как контроль/экзамен; диагностика — персонализация маршрута.
