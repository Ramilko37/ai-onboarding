# Demo-First Roadmap: MVP AI-Агента Онбординга к 25 июня 2026

## Summary

Цель: к четвергу, **25 июня 2026**, иметь рабочий MVP для показа заказчикам. Это demo-first прототип, а не production-система: он должен убедительно показать ценность AI-слоя поверх онбординга.

Главный demo flow: **новичок получает персональный маршрут → задаёт вопросы AI-ассистенту → получает ответы по базе знаний с источниками → закрывает задачи → HR/руководитель видит прогресс, блокеры и эскалации**.

Фокус MVP:

- AI-ассистент по curated базе знаний.
- Персональный onboarding-маршрут по роли.
- Чек-лист задач с прогрессом.
- HR/manager dashboard.
- Эскалация к человеку.
- Моки интеграций вместо реальных HRIS/LMS/SSO.

## Key Implementation Changes

- Создать web-прототип с тремя основными зонами: экран новичка, чат AI-ассистента, HR/manager dashboard.
- Использовать seed-данные для 2–3 ролей: например `Support Specialist`, `Sales Manager`, `Developer`.
- Подготовить локальную базу знаний для демо: FAQ, политики, первый день, доступы, обучение, контакты HR/наставника.
- Реализовать ответы ассистента только по разрешённым источникам; каждый уверенный ответ должен показывать источник.
- Для неизвестных/чувствительных вопросов использовать fallback: “Я не уверен, передам вопрос HR/наставнику”.
- Реализовать onboarding tasks со статусами: `todo`, `in_progress`, `done`, `blocked`.
- В dashboard показать: прогресс новичков, заблокированные задачи, частые вопросы, открытые эскалации.
- Все внешние интеграции заменить моками: HRIS, LMS, SSO, service desk, calendar, messaging.

## Test Plan

- Новичок открывает маршрут и видит задачи по этапам адаптации.
- Новичок задаёт вопрос “Что мне сделать в первый день?” и получает ответ с источником.
- Новичок задаёт вопрос “Как получить доступ к CRM?” и получает инструкцию с источником.
- Новичок задаёт чувствительный или неизвестный вопрос, агент создаёт эскалацию к HR/наставнику.
- Новичок меняет статус задачи, HR dashboard обновляет прогресс.
- HR видит минимум одного новичка с блокером и одну открытую эскалацию.
- Демо запускается без ручной настройки и проходит за 5–7 минут.

## Assumptions

- Текущий репозиторий пустой, поэтому MVP можно проектировать без ограничений существующей архитектуры.
- При отсутствии выбранного стека использовать самый быстрый web-stack команды; по умолчанию: frontend web app, локальные seed JSON/TS данные, mock API, hosted LLM или имитация retrieval для демо.
- Реальные интеграции, production security, SSO, полноценный LMS/HRIS sync и кадровые решения не входят в MVP до четверга.
- Главная метрика готовности: не полнота продукта, а убедительный end-to-end показ ценности заказчику.
